No crear commits, no crear branches, no hacer push. Solo aplicar los cambios en los archivos.

**Why:** El usuario prefiere controlar el flujo de git manualmente.

**How to apply:** En cualquier tarea de fix o feature, terminar en cuanto los archivos estén modificados. No proponer ni ejecutar ningún comando git.
