When Israel asks for English copy to explain work to his manager, follow this framing:

- **Include:** that the issue was a speed bump, how it was identified (e.g. inspecting API requests), and what benefits the fix brings
- **Omit:** who identified the problem — the manager already knows, no need to repeat it

**Why:** Israel's manager surfaced the problem; restating that in the copy is redundant and feels off to Israel.

**How to apply:** In any "explain this to a non-technical manager" request, lead with the speed bump framing, move to identification method, then benefits — skip attribution entirely.
